Text Files – Reading & Writing

Persistent data storage using Python's built-in file-handling capabilities.

What is a File?

A file is a named location on a storage device that stores related data permanently. Unlike variables that lose their values when a program terminates, files provide persistent storage. Python treats files as sequences of bytes or characters.

There are two main categories:

  • Text Files – Store data as human-readable characters (e.g., .txt, .csv, .py). Each line ends with a newline character \n.
  • Binary Files – Store data as raw bytes (e.g., .jpg, .mp3, .exe). Not human-readable.

The open() Function

The built-in open() function is used to open a file and returns a file object (also called a file handle). The general syntax is:

file_obj = open(filename, mode, encoding=None)

Errors and Exceptions

Understanding what goes wrong — and why — in Python programs.

Types of Errors in Python

Python has three fundamental error categories:

  • Syntax Errors – Detected by the Python parser before program execution begins. The code is structurally incorrect (e.g., missing colon, unmatched parenthesis).
  • Runtime Errors (Exceptions) – Occur during program execution when something unexpected happens (e.g., dividing by zero, accessing a missing file).
  • Logical Errors – The program runs without crashing but produces incorrect output due to a flaw in the algorithm.

What is an Exception?

An exception is an event that occurs during program execution that disrupts the normal flow. When Python encounters an error, it creates an exception object containing information about the error — its type, message, and traceback. If this object is not handled, the program terminates abruptly with a traceback.

Python uses a hierarchical exception class system. All exceptions derive from the base class BaseException, and most user-level exceptions inherit from Exception.

⚠️ Unhandled exceptions crash your program! Always handle exceptions that are predictable in your program flow.

Handling Exceptions

Using try, except, else, finally, and raise to manage errors gracefully.

The try…except Block

The try statement lets you test a block of code for errors. The except block lets you handle the error:

  • Python attempts to execute the try block.
  • If an exception occurs, execution jumps to the matching except block.
  • If no exception occurs, the except block is skipped.
  • Multiple except clauses can handle different exception types.

Full Syntax

try:
    # risky code
except ExceptionType1:
    # handler for type 1
except ExceptionType2 as e:
    # handler for type 2 (e holds the exception object)
except (Type3, Type4):
    # handler for multiple types
except Exception as e:
    # catch-all for any exception
else:
    # runs ONLY if no exception occurred
finally:
    # ALWAYS runs, regardless of exception or not

try:
    num = int(input("Enter a number: "))

except ValueError:

except ZeroDivisionError:
    print("Error: Cannot divide by zero!")

Example 3 — Multiple exception handlers multiple 
def safe_divide(a, b):
    try:
        a = int(a)
        b = int(b)
        result = a / b
    except (ValueError, TypeError) as e:
        print(f"Type/Value error: {e}")
        result = None
    except ZeroDivisionError:
        print("Cannot divide by zero!")
        result = None
    except Exception as e:
        print(f"Unexpected error: {e}")
        result = None
    return result

print(safe_divide(10, 0))     # → Cannot divide by zero!, None
print(safe_divide("a", 2))    # → Type/Value error: ..., None

# Define custom exception by subclassing Exception
class InsufficientFundsError(Exception):

def read_file_safe(filename):

Format Operator

Embedding values inside strings — Python's multiple approaches.

What is the Format Operator?

The format operator (%) is Python's C-style string formatting mechanism. It allows you to embed values into strings at runtime. Python now offers multiple string formatting methods, each with advantages:

  • % operator – Old-style (C-like), still widely used.
  • str.format() – More powerful and readable.
  • f-strings (Python 3.6+) – Most modern, fastest, and most readable. ⭐
  • Template strings – Simpler substitution via string.Template.

Command Line Arguments

Passing inputs to Python scripts from the terminal.

What are Command Line Arguments?

Command line arguments (also called CLI arguments) are values passed to a script when it is invoked from the terminal or command prompt, after the script name. They allow the same script to behave differently based on external inputs without modifying the source code.

Example: python script.py arg1 arg2 --verbose

Python provides two main ways to handle CLI arguments:

  • sys.argv – Simple list of all arguments as strings.
  • argparse module – Professional, feature-rich argument parsing with help text, types, defaults, and flags.

sys.argv

sys.argv is a list in the sys module that contains the command-line arguments. sys.argv[0] is always the script name itself. Arguments start from sys.argv[1]. All values are strings — you must convert types manually.

Example 1 — sys.argv basics sys.argv 
# File: greet.py
import sys

print(f"Script name: {sys.argv[0]}")
print(f"Total args: {len(sys.argv) - 1}")

if len(sys.argv) < 2:
    print("Usage: python greet.py <name> [age]")
    sys.exit(1)   # exit with error code

name = sys.argv[1]
age  = sys.argv[2] if len(sys.argv) > 2 else "unknown"

print(f"Hello, {name}! Age: {age}")

# Run: python greet.py Alice 22
Terminal: python greet.py Alice 22

Total args: 2

Example 2 — sys.argv with numeric operations sys.argv 
# File: calculator.py
import sys

if len(sys.argv) != 4:
    print("Usage: python calculator.py num1 operator num2")
    print("Example: python calculator.py 10 + 5")
    sys.exit(1)

try:
    a  = float(sys.argv[1])
    op = sys.argv[2]
    b  = float(sys.argv[3])

    print("Error: Arguments must be numbers.")

ops = {'+': a+b, '-': a-b, '*': a*b, '/': a/b if b != 0 else "undefined"}

if op not in ops:
    print(f"Unknown operator: {op}. Use +, -, *, /")
    sys.exit(1)

print(f"{a} {op} {b} = {ops[op]}")

Terminal: python calculator.py 15 * 4
15.0 * 4.0 = 60.0

Modules

Organizing code into reusable, importable files.

What is a Module?

A module is simply a Python file (ending in .py) that contains definitions of functions, classes, and variables. Modules allow you to organize, reuse, and share code across multiple programs. Any Python file can be imported as a module.

Python has three types of modules:

  • Built-in modules – Pre-installed with Python (e.g., math, os, sys, random, datetime).
  • Third-party modules – Installed via pip (e.g., numpy, pandas, requests).
  • User-defined modules – Your own .py files.

Packages

Organizing multiple modules into structured directory hierarchies.

What is a Package?

A package is a directory that contains multiple Python modules (files) and a special file called __init__.py. The __init__.py tells Python that the directory should be treated as a package, not just a folder. Packages allow you to organize large codebases into logical namespaces.

  • A package is a directory containing __init__.py.
  • A sub-package is a package inside another package.
  • The __init__.py can be empty or contain initialization code.
  • Python 3.3+ also supports namespace packages (without __init__.py), but using __init__.py is still best practice.

Example 1 — Creating a package (school package) create package 
# ─── school/__init__.py ───────────────────────
"""School management package."""
from .student import Student
from .teacher import Teacher
VERSION = "1.0.0"

# ─── school/student.py ────────────────────────
class Student:
    def __init__(self, name, grade):
        self.name  = name
        self.grade = grade
    def __repr__(self):
        return f"Student({self.name}, {self.grade})"

# ─── school/teacher.py ────────────────────────
class Teacher:
    def __init__(self, name, subject):
        self.name    = name
        self.subject = subject
    def introduce(self):
        return f"I'm {self.name}, I teach {self.subject}."

# ─── main.py ──────────────────────────────────
import school                   # import the package
from school import Student, Teacher

Example 2 — Nested sub-packages and imports sub-package 
# ─── Package structure ─────────────────────────
# myapp/
# ├── __init__.py
# └── utils/
#     ├── __init__.py
#     ├── strings.py
#     └── numbers.py

# ─── myapp/utils/strings.py ───────────────────
def capitalize_all(words):
    return [w.capitalize() for w in words]

def reverse_string(s):
    return s[::-1]

# ─── myapp/utils/numbers.py ───────────────────
def is_prime(n):
    if n < 2: return False
    for i in range(2, int(n**0.5)+1):
        if n % i == 0: return False
    return True

# ─── Using the sub-package ────────────────────
from myapp.utils import strings, numbers
from myapp.utils.strings import reverse_string

print(strings.capitalize_all(["hello", "world"]))  # → ['Hello', 'World']
print(reverse_string("Python"))                      # → nohtyP
print([n for n in range(2, 20) if numbers.is_prime(n)])
# → [2, 3, 5, 7, 11, 13, 17, 19]

Example 4 — __init__.py controlling public API with __all__ __all__ 
# ─── shapes/__init__.py ───────────────────────
# __all__ controls what is exported with 'from shapes import *'
__all__ = ["Circle", "Rectangle"]   # Triangle is hidden from *

from .circle    import Circle
from .rectangle import Rectangle
from .triangle  import Triangle      # available but not in __all__

# ─── usage ────────────────────────────────────
from shapes import *   # imports Circle and Rectangle only
from shapes import Triangle   # explicit import still works

c = Circle(5)
r = Rectangle(4, 6)
t = Triangle(3, 4, 5)

print(c.area())    # → 78.539...
print(r.area())    # → 24
print(t.perimeter())  # → 12

Absolute vs Relative Imports

TypeSyntaxWhen to Use
Absolutefrom myapp.utils import helperFrom the top-level package. Most reliable.
Relativefrom .utils import helperWithin the same package. Single dot = current package.
Relative parentfrom ..models import UserDouble dot = parent package.

Best practice: Use absolute imports whenever possible. Use relative imports only inside packages to avoid circular import issues.

💡 Virtual Environments: Always create a virtual environment (python -m venv venv) before installing packages for a project. This keeps dependencies isolated and avoids conflicts between projects.
